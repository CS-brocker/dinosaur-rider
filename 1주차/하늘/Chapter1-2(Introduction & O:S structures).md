## 강의 요약

- 컴퓨터: 정보를 처리하는 기계
    - 계산 가능성: 튜링 머신으로 계산 가능함
        - halting problem(정지 문제): 프로그램의 멈춤 여부를 알 수 있는가? 에 대한 앨런 튜링의 정리
        ```
        결과를 예측하는 프로그램을 만들 수 있다면, 그걸 거꾸로 행동하는 프로그램 또한 만들 수 있기 때문에 완벽히 예측 가능한 프로그램은 만들 수 없다.
        ```    
    - 범용성: NAND 게이트로 NOT. AND, OR 모두 계산 가능
- 정보: 불확실한 것을 측정해서 그 양을 수치적으로 표현한 것
    - ex) 동젼은 두가지 선택지가 있는데 수식으로 계산하면 아래와 같다 (1bit 즉 2가지 선택지가 있는 것)
        ```
        $$
        -\log_2{(2^{-1})} = 1 (bit)
        $$
        ```
        
- 앨런 튜링: 튜링 머신 만든 사람
    - 튜링 머신: 튜링 머신(응용 프로그램) - 유니버설 튜링 머신(운영체제) - 헤드(CPU) - 테이프(메모리)
        
        → 기본 컴퓨터 구조 패러다임을 제시함
        
- 폰 노이만
    - CPU가 메모리로부터 데이터를 Fetch하고 decode해서 ALU에게 execute 하는 패러다임을 제시
    - 폰노이만 아키텍쳐
        1. fetch: PC명령어를 읽어서 IR(Instruction Register)에 저장
        2. decode: CU가 IR 안의 명령어를 해석
        3. execute: ALU가 명령어를 실행
- 프로그램: 하드웨어에게 task를 행하게 하는 명령어의 집합
- 프로세스: 현재 실행 중인 프로그램
- 운영체제: 응용프로그램이 하드웨어를 다룰 수 있도록 해주는 인터페이스
- kernel(열매, 씨앗..): 운영체제의 핵심
- bootstrap: 컴퓨터가 켜질 때 HDD에 있는 OS를 메모리에 로드해주는 프로그램. ROM에 존재
- Interrupt: CPU가 하던 일을 멈추고 즉시 처리해야 하는 경우 버스를 통해 발생하는 신호.
- 대칭 멀티 프로세서: 여러 CPU가 하나의 메모리를 읽는 구조
- 멀티 코어 디자인: 하나의 CPU에 여러 코어가 존재하는 것(register, cache는 코어마다 존재)
- 멀티 프로그래밍: CPU효율을 높이기 위해 메모리에 동시에 여러 프로그램을 올려두는 것
- 멀티 프로세싱: CPU에서 여러 job을 드나들며 동시 실행처럼하는 것처럼 동작하는 것
- CPU 스케줄링: 멀티 프로세싱 환경에서 어떤 job을 돌릴지 선택하는 것
- Virtualization(가상화): VMM(Virtual Machine Manager) 하나의 컴퓨터에서 여러 개의 다른 환경이 돌아가게 하는 기술. os도 결국 프로그램이기 때문에 멀티 프로그래밍이 가능하다면 가상화도 가능함. 하드웨어에 VMM을 이용하여 여러 개의 VM(별도 kernel을 가짐)을 올려서 돌림
- 시스템콜: 응용프로그램에서 민감한 OS의 작업(파일 오픈, 메모리 할당 요청, 인터넷 패킷 요청같은)을 요청하기 위한 명령어 (ex. read(), write(), fork(), exec(), socket())
    - swift의 경우에는 [Swift → 런타임/표준 라이브러리(Foundation) → C 함수 → POSIX 시스템콜(UNIX 계열에서 쓰는 os 인터페이스) → 커널]
