## 1. GPU를 만들 때 왜 부동 소수점 연산 회로가 사용되는지

|  | 부동소수점 | 실수 |
| --- | --- | --- |
| 표현 범위 | 매우 넓음 | 제한적 |
| 오차 | 존재 (근사값) | 없음 |
| 메모리 사용량 | 큼 (4~8byte) | 작읍 (1~ 4byte) |
| 정밀도(얼마나 작은 단위까지 구분 가능하냐) | 높음 | 낮음 |

오차를 조금 감수하더라도 넓은 표현 범위와 수학적 자유도를 택한 것

GPU의 핵심은 병렬 수치 연산

- GPU가 다루는 데이터(그래픽, 물리 시뮬레이션, 딥러닝)는 정수로는 표현이 불가능함.
  너무 크거나 작은 실수 범위를 필요로 하기 때문에 부동소수점의 정밀도가 중요함.

### 정수 연산

- 모든 연산이 정확하게 떨어짐.
- 표현 가능한 범위가 한정적
    - 32비트 정수 → -21억 ~ + 21억

### 부동소수점 연산

실수를 표현하기 위한 IEEE 754 표준 방식

- 아주 작은 값(0.00000012)부터 아주 큰 값 (1e38)까지 표현 가능
- 근사치가 되는 이유
    - 2진수로 표현해야 하기 때문

        ```
        0.1를 2진수로 표현하면
        0.1₁₀ = 0.00011001100110011...(무한 반복)₂
        
        GPU나 CPU 에서는 이걸 32비트 안에서 끊어 저장
        0.1 ≈ 0.10000000149011612 -> 정확한 0.1이 아님
        ```

## 2. P2P 네트워크 구조가 어떻게 블록 체인에서 사용되는지

### P2P 네트워크

- 중앙 서버가 없음. → 각 노드가 서버이자 클라이언트 역할
- 분산성 → 한 노드가 죽어도 전체 네트워크는 계속 작동
- 데이터 복제 → 여러 노드가 동일한 데이터를 공유

### 블록체인은 P2P 네트워크 위에 구축된 탈중앙화 데이터베이스이다.

- 모든 노드는 블록체인의 전체 복사본을 가지고 있음.
- 서로 직접 통신하여 트랜잭션과 블록을 주고 받음.
- 합의 알고리즘(PoW, PoS 등)을 통해 동일한 상태를 유지함.

블록체인 = P2P 네트워크 + 합의 알고리즘 + 암호학

### P2P의 필요성

- 중앙 서버 없이 모든 노드가 독립적으로 작동해야 함.
- 각 노드가 복사본을 검증하므로 위변조가 어려움
- 일부 노드가 중단돼도 네트워크 전체가 유지됨.
- 특정 기관이 거래를 차단하거나 변경할 수 없음

→ 중앙 기관 없이도 신뢰가 유지되며 특정 기관의 개입으로 거래가 오염되는 것을 막을 수 있다.
