# 운영체제 면접 대비 질문 (프로세스 & IPC)

## 💡 면접 질문 3개

### Q1. Context Switching이 발생할 때 비용이 드는 이유는 무엇인가요?
PCB 저장과 복원 과정에서 CPU 레지스터, 프로그램 카운터, 캐시 미스 등이 발생하기 때문입니다.  
같은 프로세스 내 커널 모드 전환은 메모리 공간이 동일하므로 비용이 적지만,  
다른 프로세스로 전환할 때는 메모리 맵과 캐시까지 전환해야 해 비용이 커집니다.

---

### Q2. Shared Memory와 Message Passing의 차이를 설명하고, 각각의 장단점을 말해보세요.
| 구분 | Shared Memory | Message Passing |
|------|----------------|----------------|
| 속도 | 빠름 (직접 접근) | 느림 (OS 개입) |
| 동기화 | 개발자가 직접 관리 | OS가 관리 |
| 구현 난이도 | 높음 | 낮음 |
| 사용 예시 | DB, 캐시 서버 | 클라이언트-서버 통신 |

---

### Q3. 소켓 통신과 RPC의 차이는 무엇인가요?
소켓은 데이터 단위의 전송만 제공하며, 클라이언트·서버가 프로토콜을 직접 정의해야 합니다.  
RPC는 함수 호출처럼 원격 프로세스 실행을 추상화하며, 내부적으로 스텁(stub)과 직렬화를 통해 소켓 통신을 래핑합니다.  
예: gRPC, Java RMI, Android Binder 등이 RPC 모델 기반입니다.
