## 요약본

📘 정리 요약

1️⃣ 프로세스 개념
	•	실행 중인 프로그램 = 프로세스
	•	OS는 프로세스 단위로 CPU, 메모리, 파일, I/O 자원을 관리
	•	하나의 프로세스는 Text / Data / Heap / Stack 영역으로 구성
	•	프로세스 상태: New → Ready → Running → Waiting → Terminated
	•	PCB(Process Control Block): 프로세스 상태, 레지스터, PC, 스케줄링 정보 등을 저장
	•	Context Switch: CPU가 다른 프로세스로 전환될 때 PCB를 저장/복원하는 과정 (비용 발생)

2️⃣ 멀티태스킹 / 멀티프로세스 / 멀티스레딩
	•	멀티태스킹: 여러 프로세스를 번갈아 실행 (CPU 스케줄링)
	•	멀티프로세스: 서로 독립된 메모리 공간을 가진 여러 프로세스 동시 실행
	•	멀티스레딩: 같은 메모리 공간(Heap, Data)을 공유하며 동작 (Stack만 분리)

3️⃣ IPC (Inter-Process Communication)

프로세스 간 데이터 전달 방법.
두 프로세스가 독립적이면 문제 없지만, 협력적(cooperating)이라면 통신이 필요함.

A. Shared Memory
	•	프로세스가 OS를 통해 공유 메모리 영역을 확보
	•	데이터 접근이 빠르지만, 동기화(뮤텍스, 세마포어)는 개발자가 직접 관리
	•	구현 예: POSIX shared memory (shm_open, ftruncate, mmap)

B. Message Passing
	•	OS가 제공하는 send/receive 추상화로 메시지 교환
	•	구현은 OS가 관리 (개발자는 시스템콜로 호출)
	•	직접 vs 간접, 동기 vs 비동기, 자동 vs 명시적 버퍼링 방식 존재
	•	직접/direct: 프로세스끼리 바로 연결
	•	간접/indirect: mailbox나 port를 통한 전달
	•	동기/synchronous: 전송·수신 완료까지 블로킹
	•	비동기/asynchronous: OS가 백그라운드로 처리

C. Pipes
	•	동일한 기기 내 단방향(또는 양방향) 통신
	•	부모-자식 프로세스 간 사용이 일반적
	•	pipe() 시스템콜로 생성

D. Sockets
	•	네트워크를 통한 프로세스 간 통신
	•	IP 주소 + Port 번호로 엔드포인트를 구분
	•	구조화되지 않은 바이트 스트림을 교환 (TCP/UDP 등)

E. RPC(Remote Procedure Call)
	•	네트워크 상의 다른 프로세스 함수를 호출하듯 실행
	•	스텁(stub)이 통신을 대리
	•	XDR로 이기종 간 데이터 포맷을 표준화
